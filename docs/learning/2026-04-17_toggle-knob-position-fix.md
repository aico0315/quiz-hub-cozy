# トグルボタンのknobずれ修正：border vs outline の違い

## 何が起きていたか

ダークモードに切り替えると、トグルボタンの丸（knob）が上下中心からずれて見える問題がありました。

ライトモードでは正常 → ダークモードにするとずれる、という現象だったため、「ダークモード時だけに適用されているCSSが原因」と特定できました。

---

## 原因：border がボタンの内側を狭める

### box-sizing: border-box とは？

Tailwind CSS（および多くのモダンなCSSフレームワーク）は、デフォルトで `box-sizing: border-box` という設定が使われています。

`border-box` とは、「`width` や `height` で指定したサイズの**中に** border（枠線）も含める」という意味です。

```
border-box の場合：
┌─────────────────────────┐  ← height: 24px の外側
│  border: 1.5px          │
│  ┌───────────────────┐  │
│  │  内側: 21px       │  │  ← 実際に使える高さは 24 - 1.5 - 1.5 = 21px
│  └───────────────────┘  │
│  border: 1.5px          │
└─────────────────────────┘
```

### ボタンとknobのサイズ関係

```css
/* ボタン */
height: 24px;

/* knob（丸） */
height: 18px;
top: 3px;  /* 上から3px */
```

ライトモード（border なし）では：
- ボタン内側の高さ = **24px**
- knob の上 = 3px、knob の高さ = 18px、下の余白 = 3px → **上下均等で中央** ✅

ダークモード（border: 1.5px あり）では：
- ボタン内側の高さ = 24 - 1.5 - 1.5 = **21px**
- knob の上 = 3px、knob の高さ = 18px、下の余白 = 0px → **下に寄ってずれる** ❌

---

## 修正方法：border を outline に変える

### border と outline の違い

| プロパティ | レイアウトへの影響 | 見た目 |
|---|---|---|
| `border` | **あり**（内側を狭める） | 枠線 |
| `outline` | **なし**（サイズに影響しない） | 枠線（ほぼ同じ見た目） |

`outline` は要素の外側に描かれるため、内側の高さが変わりません。

### 修正前

```css
.themeToggle.darkMode {
  background: #3730a3;
  border: 1.5px solid #4338ca;  /* 内側が狭まる → knobがずれる */
}
```

### 修正後

```css
.themeToggle.darkMode {
  background: #3730a3;
  outline: 1.5px solid #4338ca;  /* レイアウト影響なし → knobが中央のまま */
}
```

---

## まとめ

- `border` はボックスの内側サイズを変える（border-box の場合）
- `outline` はサイズに影響しない → 位置がずれない
- 「枠線を追加したら中の要素の位置がおかしくなった」という場合、`border` → `outline` への変更が有効な解決策になることがある
